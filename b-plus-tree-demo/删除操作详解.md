# B+树删除操作详解

## 概述

B+树的删除操作比插入操作更复杂，因为删除可能导致节点"下溢"（键数太少），需要额外的处理来维持B+树的平衡性质。

## 删除操作的核心挑战

### 1. 下溢问题

在B+树中，除了根节点外，每个节点都必须满足：
- **最少键数**：`⌈order/2⌉ - 1`
- **最多键数**：`order - 1`

当删除操作导致节点的键数少于最少要求时，就发生了**下溢**。

**例如（order = 3）：**
```
最少键数 = ⌈3/2⌉ - 1 = 2 - 1 = 1
最多键数 = 3 - 1 = 2

如果一个节点有1个键，再删除一个键就会变成0个键 → 下溢！
```

### 2. 删除策略

处理下溢有三种策略（按优先级）：

```
1. 从左兄弟借键
   ↓ (如果左兄弟键数充足)
   
2. 从右兄弟借键
   ↓ (如果右兄弟键数充足)
   
3. 合并节点
   ↓ (当兄弟节点都没有多余的键可借时)
```

## 删除操作的完整流程

### 步骤1：找到并删除键

```java
public boolean delete(int key) {
    // 1. 找到包含该键的叶子节点
    BPlusTreeNode leaf = findLeafNode(key);
    
    // 2. 在叶子节点中查找并删除键
    int keyIndex = -1;
    for (int i = 0; i < leaf.getKeyCount(); i++) {
        if (leaf.getKey(i) == key) {
            keyIndex = i;
            break;
        }
    }
    
    // 3. 如果键不存在，返回false
    if (keyIndex == -1) {
        return false;
    }
    
    // 4. 删除键
    leaf.removeKey(keyIndex);
    
    // 5. 后续处理下溢...
}
```

### 步骤2：检查下溢

```java
int minKeys = getMinKeys();  // ⌈order/2⌉ - 1
if (leaf.getKeyCount() < minKeys && leaf != root) {
    handleUnderflow(leaf);  // 处理下溢
}
```

### 步骤3：处理下溢

```java
private void handleUnderflow(BPlusTreeNode node) {
    BPlusTreeNode parent = node.parent;
    
    // 找到node在父节点的子节点列表中的位置
    int index = findChildIndex(parent, node);
    
    // 策略1：尝试从左兄弟借键
    if (index > 0) {
        BPlusTreeNode leftSibling = parent.getChild(index - 1);
        if (leftSibling.getKeyCount() > getMinKeys()) {
            borrowFromLeft(node, leftSibling, index - 1);
            return;
        }
    }
    
    // 策略2：尝试从右兄弟借键
    if (index < parent.children.size() - 1) {
        BPlusTreeNode rightSibling = parent.getChild(index + 1);
        if (rightSibling.getKeyCount() > getMinKeys()) {
            borrowFromRight(node, rightSibling, index);
            return;
        }
    }
    
    // 策略3：合并节点
    if (index > 0) {
        // 与左兄弟合并
        BPlusTreeNode leftSibling = parent.getChild(index - 1);
        merge(leftSibling, node, index - 1);
    } else {
        // 与右兄弟合并
        BPlusTreeNode rightSibling = parent.getChild(index + 1);
        merge(node, rightSibling, index);
    }
}
```

## 三种核心操作详解

### 1. 从左兄弟借键

**场景：**
```
父节点：[10, 20]
        /    |    \
左兄弟：[5, 6]  当前：[]  右兄弟：[25, 30]
         ↑ 键数充足    ↑ 下溢了！
```

**操作步骤：**
```java
private void borrowFromLeft(BPlusTreeNode node, BPlusTreeNode leftSibling, int parentIndex) {
    BPlusTreeNode parent = node.parent;
    
    if (node.isLeaf) {
        // 1. 从左兄弟移除最大的键
        int borrowedKey = leftSibling.removeKey(leftSibling.getKeyCount() - 1);
        
        // 2. 插入到当前节点的最前面
        node.insertKey(0, borrowedKey);
        
        // 3. 更新父节点中的分隔键
        parent.setKey(parentIndex, node.getKey(0));
    }
}
```

**结果：**
```
父节点：[6, 20]         ← 分隔键更新为6
        /    |    \
左兄弟：[5]  当前：[6]  右兄弟：[25, 30]
         ↑ 借出一个     ↑ 借入成功！
```

### 2. 从右兄弟借键

**场景：**
```
父节点：[10, 20]
        /    |    \
左兄弟：[5]   当前：[]  右兄弟：[25, 30]
         ↑ 下溢了！    ↑ 键数充足
```

**操作步骤：**
```java
private void borrowFromRight(BPlusTreeNode node, BPlusTreeNode rightSibling, int parentIndex) {
    BPlusTreeNode parent = node.parent;
    
    if (node.isLeaf) {
        // 1. 从右兄弟移除最小的键
        int borrowedKey = rightSibling.removeKey(0);
        
        // 2. 追加到当前节点的最后面
        node.keys.add(borrowedKey);
        
        // 3. 更新父节点中的分隔键
        parent.setKey(parentIndex, rightSibling.getKey(0));
    }
}
```

**结果：**
```
父节点：[25, 20]        ← 分隔键更新为25
        /    |    \
左兄弟：[5]  当前：[25] 右兄弟：[30]
         ↑ 借入成功！   ↑ 借出一个
```

### 3. 合并节点

**场景：**
```
父节点：[10, 20]
        /    |    \
左兄弟：[5]   当前：[]  右兄弟：[25]
         ↑ 无法借      ↑ 也无法借（都只有最少键数）
```

**操作步骤（叶子节点合并）：**
```java
private void merge(BPlusTreeNode leftNode, BPlusTreeNode rightNode, int parentIndex) {
    BPlusTreeNode parent = leftNode.parent;
    
    if (leftNode.isLeaf) {
        // 1. 将rightNode的所有键追加到leftNode
        leftNode.keys.addAll(rightNode.keys);
        
        // 2. 维护叶子节点链表
        leftNode.nextLeaf = rightNode.nextLeaf;
        
        // 3. 从父节点移除分隔键
        parent.removeKey(parentIndex);
    }
    
    // 4. 从父节点移除rightNode的引用
    parent.removeChild(parentIndex + 1);
}
```

**结果：**
```
父节点：[20]              ← 移除了分隔键10
        /    \
合并节点：[5, 25]  右兄弟：[30]
         ↑ 合并成功，键数充足
```

## 内部节点的删除

对于内部节点的删除，逻辑类似，但需要额外处理子节点：

### 内部节点借键

```java
// 从左兄弟借键（内部节点）
private void borrowFromLeft(BPlusTreeNode node, BPlusTreeNode leftSibling, int parentIndex) {
    BPlusTreeNode parent = node.parent;
    
    // 1. 从父节点降下一个键到当前节点
    int parentKey = parent.getKey(parentIndex);
    node.insertKey(0, parentKey);
    
    // 2. 从左兄弟借最大的键上升到父节点
    int borrowedKey = leftSibling.removeKey(leftSibling.getKeyCount() - 1);
    parent.setKey(parentIndex, borrowedKey);
    
    // 3. 从左兄弟借最右边的子节点
    BPlusTreeNode borrowedChild = leftSibling.removeChild(leftSibling.children.size() - 1);
    node.insertChild(0, borrowedChild);
    borrowedChild.parent = node;
}
```

### 内部节点合并

```java
// 合并内部节点
private void merge(BPlusTreeNode leftNode, BPlusTreeNode rightNode, int parentIndex) {
    BPlusTreeNode parent = leftNode.parent;
    
    // 1. 从父节点降下分隔键到leftNode
    int parentKey = parent.removeKey(parentIndex);
    leftNode.keys.add(parentKey);
    
    // 2. 将rightNode的所有键和子节点追加到leftNode
    leftNode.keys.addAll(rightNode.keys);
    leftNode.children.addAll(rightNode.children);
    
    // 3. 更新子节点的父引用
    for (BPlusTreeNode child : rightNode.children) {
        child.parent = leftNode;
    }
    
    // 4. 从父节点移除rightNode
    parent.removeChild(parentIndex + 1);
}
```

## 根节点的特殊处理

### 根节点收缩

当根节点的所有键都被删除，且根节点不是叶子节点时：

```java
if (parent == root && parent.isEmpty()) {
    // 树的高度减1
    if (!parent.isLeaf && parent.children.size() > 0) {
        root = parent.getChild(0);
        root.parent = null;
    }
}
```

**示例：**
```
删除前：
    [10]
   /    \
 [5]    [10, 20]

删除10和20后：
    []           ← 根节点为空
   /
 [5]

收缩后：
  [5]            ← 新的根节点
```

## 删除操作示例

### 完整示例（order = 3）

```
初始状态：
        [12]
       /    \
   [6,10]   [20]
   /  | \   /  \
 [5] [6,7] [10] [12,17] [20,30]

删除7：
        [12]
       /    \
   [6,10]   [20]
   /  | \   /  \
 [5] [6] [10] [12,17] [20,30]
      ↑ 不引发下溢（还有1个键）

删除5：
        [12]
       /    \
    [10]    [20]
    /  \    /  \
  [6]  [10] [12,17] [20,30]
   ↑ 从右兄弟借位（6提升到父节点，10降下来）

删除10：
      [12, 20]
     /   |    \
   [6] [12,17] [20,30]
    ↑ 合并操作，树高度从3降到2
```

## 关键要点

1. **删除总是在叶子节点进行**
   - B+树的所有数据都在叶子节点
   - 内部节点只起索引作用

2. **下溢处理优先级**
   - 先尝试借位（保持节点数量不变）
   - 借位失败才合并（减少节点数量）

3. **维护不变量**
   - 删除后，所有叶子节点仍在同一层
   - 键的有序性保持不变
   - 每个节点的键数满足最少/最多要求

4. **级联处理**
   - 合并可能导致父节点下溢
   - 需要递归向上处理
   - 最坏情况可能到达根节点，导致树高减1

## 复杂度分析

- **时间复杂度**：`O(log n)`
  - 查找：`O(log n)`
  - 删除：`O(1)`（叶子节点内）
  - 借位/合并：`O(log n)`（可能需要向上递归）

- **空间复杂度**：`O(1)`
  - 只需要常数个额外指针
