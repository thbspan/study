## 理解B+树的核心概念

### 什么是B+树？

B+树是一种多路搜索树，是B树的变种。它的主要特点是：
- 所有数据记录都存储在叶子节点中
- 非叶子节点只存储索引信息
- 叶子节点通过指针连接成链表

### 为什么使用B+树？

1. **适合磁盘存储**：节点大小可以与磁盘块对齐，减少I/O次数
2. **范围查询高效**：叶子节点的链表结构使范围查询非常简单
3. **查询性能稳定**：所有查询都需要走到叶子节点，性能一致
4. **顺序访问友好**：可以通过叶子节点链表快速遍历所有数据

## 代码结构解析

### 节点类 (BPlusTreeNode.java)

```java
public class BPlusTreeNode {
    boolean isLeaf;                    // 是否为叶子节点
    List<Integer> keys;                // 键列表
    List<BPlusTreeNode> children;      // 子节点列表（非叶子节点）
    BPlusTreeNode nextLeaf;           // 下一个叶子节点（叶子节点）
    BPlusTreeNode parent;             // 父节点
}
```

**关键字段说明：**
- `isLeaf`: 区分叶子节点和内部节点
- `keys`: 存储的键值，保持有序
- `children`: 子节点引用，数量 = keys.size() + 1
- `nextLeaf`: 连接所有叶子节点，支持范围查询

## 操作详解

### 1. 插入操作 (insert)

**步骤：**
1. 找到合适的叶子节点
2. 按序插入键值
3. 检查是否溢出（keys.size() > order - 1）
4. 如果溢出，执行分裂操作

**示例（order = 3）：**
```
插入 10: [10]
插入 20: [10, 20]
插入 5:  分裂 -> 
       [10]
      /    \
   [5]    [10, 20]
```

### 2. 查找操作 (search)

**步骤：**
1. 从根节点开始
2. 在当前节点中找到合适的子节点
3. 移动到子节点
4. 重复2-3直到叶子节点
5. 在叶子节点中查找目标键

**示例（查找 12）：**
```
      [10, 12]
     /    |    \
  [5,6] [10] [12,20]
  
从根节点开始 -> 12 >= 12，选择最右子节点 -> 在[12,20]中找到12
```

### 3. 删除操作 (delete)

删除操作比插入更复杂，需要处理下溢情况。

**步骤：**
1. 找到包含该键的叶子节点
2. 删除键
3. 检查是否下溢（keys.size() < minKeys）
4. 如果下溢，执行以下策略（按优先级）：
   - 从左兄弟借键
   - 从右兄弟借键
   - 合并节点

**示例（order = 3）：**
```
初始：      [12]
           /    \
       [6,10]   [20]
       /  | \   /  \
     [5] [6,7] [10] [12,17] [20,30]

删除7：    [12]
           /    \
       [6,10]   [20]
       /  | \   /  \
     [5] [6] [10] [12,17] [20,30]
          ↑ 不引发下溢（还有1个键）
          
删除5：    [12]
           /    \
        [10]    [20]
        /  \    /  \
      [6]  [10] [12,17] [20,30]
       ↑ 从右兄弟借位
```

**借位操作：**
```java
// 从左兄弟借键：借最大的键，放到当前节点最前面
borrowFromLeft(node, leftSibling, index);

// 从右兄弟借键：借最小的键，放到当前节点最后面  
borrowFromRight(node, rightSibling, index);
```

**合并操作：**
```java
// 当兄弟节点都没有多余的键可借时，执行合并
merge(leftNode, rightNode, parentIndex);
```

详细删除操作说明请查看 [删除操作详解.md](删除操作详解.md)

### 4. 分裂操作 (split)

**叶子节点分裂：**
```java
private void splitLeaf(BPlusTreeNode leaf) {
    // 1. 计算中点
    int mid = leaf.getKeyCount() / 2;
    
    // 2. 创建新节点，移动后半部分
    BPlusTreeNode newLeaf = new BPlusTreeNode(true);
    // 移动键...
    
    // 3. 维护叶子节点链表
    newLeaf.nextLeaf = leaf.nextLeaf;
    leaf.nextLeaf = newLeaf;
    
    // 4. 将新节点的最小键提升到父节点
    insertIntoParent(leaf, newLeaf.getKey(0), newLeaf);
}
```


## 阶数(order)的影响

阶数决定了每个节点可以存储的最大键数：

- **order = 3**: 每个节点最多 2 个键
- **order = 4**: 每个节点最多 3 个键  
- **order = m**: 每个节点最多 m-1 个键

**阶数选择：**
- 阶数小：树更高，分裂更频繁
- 阶数大：树更矮，节点利用率更高

## 常见问题

**Q: B+树和B树有什么区别？**
A: B树的所有节点都存储数据，而B+树只有叶子节点存储数据。B+树的叶子节点通过指针连接，B树没有这个特性。

**Q: 为什么B+树适合数据库索引？**
A: 因为B+树的高度低（减少磁盘I/O），叶子节点的链表结构支持高效的范围查询，且所有查询性能一致。

**Q: 阶数应该如何选择？**
A: 实际应用中，阶数通常根据磁盘块大小和键的大小来计算，使得一个节点刚好填满一个磁盘块。
