# B+树持久化存储实现说明

## 概述

本项目实现了类似MySQL InnoDB存储引擎的基于数据页的B+树持久化存储机制。通过页级别的读写操作、缓冲池管理和序列化技术，实现了数据的持久化存储。

## 数据页结构设计

### 文件格式

```
[文件头 64字节] [页0 4096字节] [页1 4096字节] [页2 4096字节] ...
```

### 文件头结构 (64字节)

| 偏移 | 大小 | 字段 | 说明 |
|------|------|------|------|
| 0 | 4字节 | 魔数 | 0x42504C54 ("BPLT") |
| 4 | 4字节 | 版本号 | 当前为1 |
| 8 | 4字节 | 根节点页号 | 根节点所在页 |
| 12 | 4字节 | 总页数 | 数据库总页数 |
| 16-63 | 48字节 | 保留 | 未来扩展 |

### 页结构 (4096字节)

```
[页头 32字节] [数据区 4064字节]
```

#### 页头结构 (32字节)

| 偏移 | 大小 | 字段 | 说明 |
|------|------|------|------|
| 0 | 1字节 | 页类型 | 0=内部节点, 1=叶子节点 |
| 1 | 2字节 | 键/记录数量 | 当前节点的键数 |
| 3 | 4字节 | 父节点页号 | 父节点所在页 |
| 7 | 4字节 | 下一个叶子页号 | 叶子节点链表 |
| 11 | 4字节 | 前一个叶子页号 | 叶子节点链表 |
| 15-31 | 17字节 | 保留 | 未来扩展 |

#### 数据区结构

**内部节点：**
```
[键1 4字节][子页号1 4字节][键2 4字节][子页号2 4字节]...[最后一个子页号 4字节]
```

**叶子节点：**
```
[键1 4字节][值长度 2字节][值数据 N字节][键2 4字节][值长度 2字节][值数据 N字节]...
```

## 核心机制

### 1. 页寻址

与传统内存B+树使用对象引用不同，持久化B+树使用**页号**来定位节点：

```java
// 内存版：使用对象引用
BPlusTreeNode child = node.getChild(i);

// 持久化版：使用页号
int childPageNum = node.getChildPageNum(i);
byte[] pageData = bufferPool.getPage(childPageNum, pageManager);
BPlusTreeNode child = pageSerializer.deserializeNode(pageData, childPageNum);
```

### 2. 缓冲池 (BufferPool)

缓冲池使用LRU(Least Recently Used)策略缓存频繁访问的页：

```java
// 获取页（缓存命中则直接返回，否则从磁盘读取）
byte[] pageData = bufferPool.getPage(pageNum, pageManager);

// 修改页后标记为脏页
bufferPool.markDirty(pageNum, modifiedPageData);

// 关闭数据库时刷新所有脏页
bufferPool.flushAll(pageManager);
```

**脏页机制：**
- 修改页时不立即写入磁盘
- 将修改的页标记为"脏页"
- 在关闭数据库或手动flush时批量写入
- 减少磁盘I/O次数

### 3. 序列化/反序列化

将内存中的节点结构转换为可存储的字节格式：

```java
// 序列化：节点 → 字节数组
byte[] pageData = pageSerializer.serializeLeafNode(
    records, pageNum, parentPageNum, nextPageNum, prevPageNum
);

// 反序列化：字节数组 → 节点
BPlusTreeNode node = pageSerializer.deserializeNode(pageData, pageNum);
```

## 与MySQL InnoDB的对比

| 特性 | 本实现 | MySQL InnoDB |
|------|--------|--------------|
| 页大小 | 4KB | 16KB（可配置） |
| 缓冲池 | 简单LRU | 复杂LRU+预读 |
| 并发控制 | 无 | MVCC+行级锁 |
| 事务支持 | 无 | ACID事务 |
| WAL日志 | 无 | Redo Log + Undo Log |
| 页分裂 | 50-50分裂 | 自适应分裂 |
| 压缩 | 无 | 页压缩 |

## 使用示例

### 基本操作

```java
// 1. 创建/打开数据库
PersistentBPlusTree tree = new PersistentBPlusTree(3);
tree.open("bplus_tree.db");

// 2. 插入记录
tree.insert(10, "张三");
tree.insert(20, "李四");
tree.insert(5, "王五");

// 3. 精确查询
Record record = tree.search(10);
System.out.println(record.getValue());  // 输出: 张三

// 4. 范围查询
List<Record> records = tree.rangeQuery(5, 20);

// 5. 删除记录
tree.delete(10);

// 6. 关闭数据库（自动保存）
tree.close();
```

### 持久化验证

```java

tree2.close();
```

## 学习要点

### 1. 为什么需要数据页？

- **减少I/O**：一次性读取/写入一块数据，而不是逐条记录
- ** locality**：相关的数据在物理上相邻，提高缓存命中率
- **管理方便**：以页为单位分配、释放、缓存

### 2. 为什么需要缓冲池？

- **性能**：内存访问比磁盘快100万倍
- **减少I/O**：合并多次修改，批量写入
- **LRU策略**：自动保留最常用的数据

### 3. 为什么需要序列化？

- **持久化**：将内存对象转换为可存储的字节
- **网络传输**：可以在不同机器间传输
- **跨平台**：字节格式与编程语言无关

### 4. 页号 vs 对象引用

```
内存B+树：                    持久化B+树：
node → [对象引用] → child     node → [页号: 5] → 磁盘页5
```

使用页号的好处：
- 不受内存地址变化影响
- 支持数据库重启
- 可以大于当前内存中的节点数
